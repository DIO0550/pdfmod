# PDF解析パイプライン（Phase 1 + 2）

> **バージョン**: 1.0
> **作成日**: 2026-03-15
> **作成者**: pdfmod開発チーム
> **ステータス**: 下書き

## 概要

`@pdfmod/core` パッケージにおけるPDF解析パイプラインのPhase 1（Graph Resolver）とPhase 2（DOM Traverser）の機能仕様書。PDFバイナリからドキュメント構造（ページ一覧・メタデータ）を構築するまでの処理を定義する。

## 背景

現在、Lexer/Tokenizer（バイト列→トークン列変換）は実装済みだが、トークン列からPDFのドキュメント構造を構築するモジュールが存在しない。xref解析・オブジェクト解決・ページツリー走査を実装することで、任意のPDFファイルからページ情報を取得可能にする。

## スコープ

**対象範囲**:
- ファイル末尾からのstartxref/%%EOF検出
- xrefテーブル解析（テキスト形式・ストリーム形式）
- `/Prev`チェーンによるxrefマージ（インクリメンタルアップデート対応）
- トレイラ辞書解析
- Token列→PdfObject変換（ObjectParser）
- インダイレクト参照のオブジェクト解決（LRUキャッシュ付き）
- オブジェクトストリーム内オブジェクトの抽出
- ページツリー再帰走査（`/Pages` → `/Page`）
- ページ属性の継承解決（MediaBox, Resources, CropBox, Rotate）
- `PdfDocument` / `PdfPage` パブリックAPI
- 循環参照検出
- 壊れたPDFへの寛容処理（フォールバックxrefスキャナ）

**対象外**:
- コンテンツストリーム解釈（Phase 3）
- フォント処理（Phase 4）
- ドキュメント書き出し（Phase 5）
- 暗号化/復号化
- ストリームフィルタ展開（FlateDecode等 — Phase 1ではxrefストリームの展開のみ）

## ユーザーストーリー

| ID | ～として | ～したい | ～のために | 優先度 |
|:---|:---------|:---------|:-----------|:-------|
| US-001 | ライブラリ利用者 | PDFファイルをUint8Arrayで読み込んでPdfDocumentオブジェクトを取得 | ページ数やメタデータにアクセスするため | 高 |
| US-002 | ライブラリ利用者 | 特定ページのMediaBoxやRotate情報を取得 | ページの寸法・向きを把握するため | 高 |

| US-004 | ライブラリ利用者 | 大きなPDFでもメモリを過剰消費せずに読み込む | ブラウザ環境でも安全に使用するため | 中 |

## 処理フロー

```
Uint8Array (PDFバイナリ)
    │
    ▼
┌─────────────────────────────┐
│ 1. ヘッダ検証 (%PDF-x.x)     │
└─────────────────────────────┘
    │
    ▼
┌─────────────────────────────┐
│ 2. EOF/startxref スキャン     │ ← 末尾1024バイトから逆方向
└─────────────────────────────┘
    │
    ▼
┌─────────────────────────────┐
│ 3. xref 解析                 │ ← テキスト or ストリーム形式
│    └─ /Prev チェーン再帰     │
└─────────────────────────────┘
    │
    ▼
┌─────────────────────────────┐
│ 4. xref マージ               │ ← 新しいものが優先
└─────────────────────────────┘
    │
    ▼
┌─────────────────────────────┐
│ 5. トレイラ → /Root 解決     │
└─────────────────────────────┘
    │
    ▼
┌─────────────────────────────┐
│ 6. カタログ辞書解析           │
└─────────────────────────────┘
    │
    ▼
┌─────────────────────────────┐
│ 7. ページツリー走査           │ ← /Pages → /Page 再帰
│    └─ 属性継承解決           │
└─────────────────────────────┘
    │
    ▼
PdfDocument (ページ一覧 + メタデータ)
```

## 非機能要件

| カテゴリ | 要件 | 目標値 |
|:---------|:-----|:-------|
| パフォーマンス | 1MBのPDFのドキュメント構造構築 | 100ms以内 |
| メモリ | 100MBのPDF読み込み時ピークメモリ | 入力サイズの2倍以内 |
| メモリ | LRUキャッシュ上限 | 設定可能（デフォルト1024エントリ） |
| 互換性 | ブラウザ | Chrome/Firefox/Safari/Edge 最新2バージョン |
| 互換性 | Node.js | 24.x |

## 用語集

| 用語 | 定義 |
|:-----|:-----|
| xref | Cross-Reference Table。オブジェクト番号→バイトオフセットの索引 |
| インダイレクト参照 | `N G R` 形式のオブジェクト間ポインタ（例: `5 0 R`） |
| MediaBox | ページの物理的な寸法を定義する矩形 `[llx lly urx ury]`。座標原点は左下 |
| オブジェクトストリーム | 複数のPDFオブジェクトを1つのストリーム内にパックした構造（PDF 1.5以降） |
| LRU | Least Recently Used。最も古い使用のエントリから追い出すキャッシュ戦略 |
| Postelの法則 | 「送信するものについては厳密に、受信するものについては寛容に」 |
