# 05. コンテンツストリームと描画オペレータ

## 1. 概要

PDFページの視覚的な構成は、すべてコンテンツストリーム（`/Contents`）内にバイナリエンコードされた命令群として記述される。PDFのオペレータ構文はPostScriptから受け継いだ**後置記法（逆ポーランド記法）**を採用している。

### 基本的な処理モデル

```
コンテンツストリームの処理:
1. ストリームデータをデコード（圧縮解除）
2. バイト列をトークン化
3. オペランド（引数）をスタックにプッシュ
4. オペレータを検出したらスタックからオペランドをポップして処理実行
```

**重要**: ループ構文（for/while）、条件分岐（if）、変数宣言は一切存在しない。絶対的なグラフィック描画命令の羅列である。

## 2. グラフィックスステートオペレータ

### 2.1 ステート管理

| オペレータ | オペランド | 説明 |
|:-----------|:-----------|:-----|
| `q` | なし | 現在のグラフィックスステートをスタックに保存 |
| `Q` | なし | スタックからグラフィックスステートを復元 |
| `cm` | a b c d e f | カレント変換行列（CTM）にアフィン変換行列を連結 |
| `w` | lineWidth | 線の太さを設定 |
| `J` | lineCap | 線端の形状を設定（0: Butt, 1: Round, 2: Square） |
| `j` | lineJoin | 線接合の形状を設定（0: Miter, 1: Round, 2: Bevel） |
| `M` | miterLimit | マイター限界値を設定 |
| `d` | dashArray dashPhase | 破線パターンを設定 |
| `ri` | intent | レンダリングインテントを設定 |
| `i` | flatness | フラットネス許容値を設定 |
| `gs` | dictName | 名前付き拡張グラフィックスステートを適用 |

### 2.2 一般的なイディオム: q/Q ペア

```
q                        % ステート保存
  1 0 0 1 100 200 cm     % 座標変換（平行移動）
  /Im1 Do                % 画像を描画
Q                        % ステート復元（変換前の状態に戻る）
```

**注意**: `q` と `Q` は必ずペアで使用しなければならない。ネストは許可される。

## 3. 座標系とアフィン変換行列

### 3.1 PDFの座標系

```
Y
^
|
|   ページの左下が原点 (0, 0)
|   X軸は右方向に正
|   Y軸は上方向に正
|
+---------> X
(0,0)
```

- デフォルトのユーザー空間は**左下原点**
- 一般的なコンピュータグラフィックス（左上原点）とは**逆**
- 単位: 1ユーザー空間単位 = 1/72インチ（デフォルト）

### 3.2 アフィン変換行列

`cm` オペレータの6つのオペランド `a b c d e f` は以下の行列を表す。

```
| a  b  0 |
| c  d  0 |
| e  f  1 |
```

#### 基本的な変換パターン

| 変換 | a | b | c | d | e | f | 例 |
|:-----|:--|:--|:--|:--|:--|:--|:---|
| 平行移動 | 1 | 0 | 0 | 1 | tx | ty | `1 0 0 1 100 200 cm` |
| 拡大縮小 | sx | 0 | 0 | sy | 0 | 0 | `2 0 0 2 0 0 cm` |
| 回転（θ） | cos θ | sin θ | -sin θ | cos θ | 0 | 0 | `0 1 -1 0 0 0 cm` (90°) |
| せん断 | 1 | shy | shx | 1 | 0 | 0 | `1 0.5 0 1 0 0 cm` |

#### 行列の連結

```
新CTM = 指定行列 × 現在のCTM

注意: 行列の乗算は非可換。適用順序が重要。
```

#### 画像配置の典型的なパターン

```
q
width 0 0 height x y cm    % 画像の位置とサイズを指定
/Im0 Do                    % 画像を描画（1x1の単位正方形から変換される）
Q
```

## 4. パス構築オペレータ

PDFは高レベルなプリミティブ命令を直接持たない。低レベルなパス構築オペレータを使用して、メモリ上にパス（軌跡）を定義する。

### 4.1 パス構築

| オペレータ | オペランド | 説明 |
|:-----------|:-----------|:-----|
| `m` | x y | カレントポイントを (x, y) に移動（サブパス開始） |
| `l` | x y | カレントポイントから (x, y) まで直線パスを追加 |
| `c` | x1 y1 x2 y2 x3 y3 | 3次ベジェ曲線。制御点 (x1,y1), (x2,y2) を経て (x3,y3) へ |
| `v` | x2 y2 x3 y3 | 3次ベジェ曲線。最初の制御点がカレントポイント |
| `y` | x1 y1 x3 y3 | 3次ベジェ曲線。最後の制御点が終点と一致 |
| `h` | なし | 現在のサブパスを閉じる（始点に直線を引く） |
| `re` | x y width height | 矩形パスを追加（唯一の高レベルプリミティブ） |

### 4.2 パス描画（ペイント）オペレータ

パスを構築した後、以下のオペレータで視覚化する。

| オペレータ | 説明 |
|:-----------|:-----|
| `S` | ストローク（パスに沿って線を描画） |
| `s` | パスを閉じてストローク（`h S` と等価） |
| `f` | フィル（非ゼロワインディング規則で塗りつぶし） |
| `F` | `f` と同じ（互換性のため） |
| `f*` | フィル（偶奇規則で塗りつぶし） |
| `B` | フィル＋ストローク（非ゼロワインディング規則） |
| `B*` | フィル＋ストローク（偶奇規則） |
| `b` | パスを閉じてフィル＋ストローク（非ゼロ） |
| `b*` | パスを閉じてフィル＋ストローク（偶奇） |
| `n` | パスを消去（クリッピングパス設定後に使用） |

### 4.3 クリッピングオペレータ

| オペレータ | 説明 |
|:-----------|:-----|
| `W` | 非ゼロワインディング規則でクリッピングパスを設定 |
| `W*` | 偶奇規則でクリッピングパスを設定 |

**注意**: クリッピングオペレータは必ずパスペイントオペレータの直前に配置する。

```
% 矩形でクリッピング
100 100 300 400 re   % 矩形パス構築
W n                   % クリッピング設定 + パス消去
% 以降の描画はこの矩形内に制限される
```

### 4.4 塗りつぶしルール

#### 非ゼロワインディング規則（Non-Zero Winding Rule）

```
パス上の任意の点から無限遠方に向かって線を引く:
- パスが左から右に横切る → カウンタ +1
- パスが右から左に横切る → カウンタ -1
- カウンタ ≠ 0 → 内部（塗りつぶし対象）
```

#### 偶奇規則（Even-Odd Rule）

```
パス上の任意の点から無限遠方に向かって線を引く:
- パスとの交差回数が奇数 → 内部（塗りつぶし対象）
- パスとの交差回数が偶数 → 外部
```

### 4.5 実装例: 矩形の描画

```
% 赤い矩形を描画
q
1 0 0 rg               % フィル色を赤に設定
0 0 1 RG               % ストローク色を青に設定
2 w                     % 線の太さ 2pt
100 100 200 150 re      % 矩形パス (x=100, y=100, w=200, h=150)
B                       % フィル＋ストローク
Q
```

### 4.6 実装例: 円の近似（ベジェ曲線）

PDFには円を直接描画する命令がないため、4つの3次ベジェ曲線で近似する。

```
% 中心 (cx, cy)、半径 r の円
% k = 0.5522847498 (ベジェ曲線の近似係数)

cx+r cy      m                           % 右端から開始
cx+r cy+k*r  cx+k*r cy+r  cx cy+r    c   % 右上 → 上
cx-k*r cy+r  cx-r cy+k*r  cx-r cy    c   % 上 → 左
cx-r cy-k*r  cx-k*r cy-r  cx cy-r    c   % 左 → 下
cx+k*r cy-r  cx+r cy-k*r  cx+r cy    c   % 下 → 右（閉じる）
S                                         % ストローク
```

## 5. 色指定オペレータ

ストローク色とフィル色は完全に独立して管理される。**大文字 = ストローク、小文字 = フィル**。

### 5.1 デバイス色空間オペレータ

| オペレータ | オペランド | 説明 |
|:-----------|:-----------|:-----|
| `G` | gray | DeviceGray ストローク色設定（0=黒, 1=白） |
| `g` | gray | DeviceGray フィル色設定 |
| `RG` | r g b | DeviceRGB ストローク色設定（各0.0〜1.0） |
| `rg` | r g b | DeviceRGB フィル色設定 |
| `K` | c m y k | DeviceCMYK ストローク色設定（各0.0〜1.0） |
| `k` | c m y k | DeviceCMYK フィル色設定 |

### 5.2 汎用色空間オペレータ

| オペレータ | オペランド | 説明 |
|:-----------|:-----------|:-----|
| `CS` | name | ストロークの色空間を設定 |
| `cs` | name | フィルの色空間を設定 |
| `SC` | c1 ... cn | ストローク色を設定（色空間に依存する成分数） |
| `sc` | c1 ... cn | フィル色を設定 |
| `SCN` | c1 ... cn [name] | ストローク色を設定（パターン対応） |
| `scn` | c1 ... cn [name] | フィル色を設定（パターン対応） |

## 6. テキスト描画オペレータ

### 6.1 テキストブロック

テキストは必ず `BT`（Begin Text）と `ET`（End Text）で囲む。

```
BT
  /F1 12 Tf              % フォント F1、サイズ 12pt
  1 0 0 1 72 720 Tm      % テキスト行列設定（位置指定）
  (Hello World) Tj       % 文字列描画
ET
```

### 6.2 テキスト状態オペレータ

| オペレータ | オペランド | 説明 |
|:-----------|:-----------|:-----|
| `Tc` | charSpace | 文字間隔（追加スペース） |
| `Tw` | wordSpace | 単語間隔（スペース文字のみに適用） |
| `Tz` | scale | 水平スケーリング（%指定、100=等倍） |
| `TL` | leading | 行間隔（テキストリーディング） |
| `Tf` | font size | フォントとサイズを設定 |
| `Tr` | render | テキストレンダリングモード（0〜7） |
| `Ts` | rise | テキストライズ（上付き・下付き用の垂直オフセット） |

### 6.3 テキスト位置決めオペレータ

| オペレータ | オペランド | 説明 |
|:-----------|:-----------|:-----|
| `Td` | tx ty | テキスト位置を移動（相対） |
| `TD` | tx ty | `Td` と同等だが、`-ty TL` も設定 |
| `Tm` | a b c d e f | テキスト行列を直接設定（絶対位置指定） |
| `T*` | なし | 次の行へ移動（`0 -TL Td` と等価） |

### 6.4 テキスト描画オペレータ

| オペレータ | オペランド | 説明 |
|:-----------|:-----------|:-----|
| `Tj` | string | 文字列を描画 |
| `TJ` | array | 文字列と位置調整値の配列を描画 |
| `'` | string | 次の行に移動して文字列を描画（`T* string Tj` と等価） |
| `"` | aw ac string | 単語間隔・文字間隔を設定して次の行に文字列を描画 |

### 6.5 TJ オペレータの詳細

`TJ` は精密なカーニング制御に使用される最も重要なテキストオペレータ。

```
[(H) 40 (ello) -20 ( W) 80 (orld)] TJ
```

- 文字列要素: そのまま描画
- 数値要素: テキスト位置を調整（**負の値 = 右に移動、正の値 = 左に移動**）
- 単位: テキスト空間の1/1000単位

#### 実装仕様
```
fn execute_tj_array(array: &PdfArray, state: &mut GraphicsState) {
    for element in array {
        match element {
            PdfObject::String(s) => {
                render_string(s, state);
            }
            PdfObject::Number(n) => {
                // テキスト位置を水平方向に調整
                // 負の値は文字間を広げる（右に移動）
                // 正の値は文字間を狭める（左に移動）
                let offset = -n / 1000.0 * state.font_size * state.horizontal_scaling / 100.0;
                state.text_matrix.translate(offset, 0.0);
            }
        }
    }
}
```

### 6.6 テキスト行列とテキスト描画位置

テキストブロック内では2つの行列が管理される。

```
テキスト行列 (Tm):
  - BT で単位行列に初期化
  - Tm オペレータで直接設定
  - Td, TD で相対移動

テキスト描画位置行列 (Trm):
  Trm = [fontSize * Tz  0           0]     [Tm]
        [0              fontSize    0]  ×
        [0              Ts          1]

  最終的な描画位置 = Trm × CTM
```

## 7. XObject 描画オペレータ

| オペレータ | オペランド | 説明 |
|:-----------|:-----------|:-----|
| `Do` | name | 名前付きXObject（画像またはフォーム）を描画 |

### 7.1 画像XObjectの描画

```
q
200 0 0 300 50 400 cm    % 幅200pt、高さ300pt、位置(50, 400)に配置
/Im0 Do                   % 画像を描画
Q
```

画像XObjectは**1×1の単位正方形**として定義されており、`cm` オペレータで適切な変換行列を設定して配置・拡大する。

### 7.2 フォームXObjectの描画

```
q
/Fm1 Do                   % フォームXObject内のストリームを実行
Q
```

フォームXObjectは独自の `/Resources` と `/BBox` を持ち、カプセル化された描画命令群として機能する。

## 8. インラインイメージオペレータ

小さな画像をストリーム内に直接埋め込むための特殊構文。

| オペレータ | 説明 |
|:-----------|:-----|
| `BI` | インラインイメージ辞書の開始 |
| `ID` | 画像データの開始（直後に改行） |
| `EI` | インラインイメージの終了 |

```
BI
  /W 16 /H 16
  /CS /RGB
  /BPC 8
  /F /Fl
ID
... (バイナリ画像データ) ...
EI
```

#### 省略形キー

| 省略形 | 完全名 |
|:-------|:-------|
| `/W` | `/Width` |
| `/H` | `/Height` |
| `/BPC` | `/BitsPerComponent` |
| `/CS` | `/ColorSpace` |
| `/F` | `/Filter` |
| `/DP` | `/DecodeParms` |
| `/IM` | `/ImageMask` |
| `/I` | `/Interpolate` |

## 9. マーク付きコンテンツオペレータ

アクセシビリティやコンテンツの論理構造を定義するためのオペレータ。

| オペレータ | オペランド | 説明 |
|:-----------|:-----------|:-----|
| `BMC` | tag | マーク付きコンテンツの開始 |
| `BDC` | tag properties | プロパティ付きマーク付きコンテンツの開始 |
| `EMC` | なし | マーク付きコンテンツの終了 |
| `MP` | tag | マーク付きコンテンツポイント |
| `DP` | tag properties | プロパティ付きマーク付きコンテンツポイント |

```
/Span <</ActualText (Hello)>> BDC
  % テキスト描画命令
EMC
```
