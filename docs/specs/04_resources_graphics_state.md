# 04. リソース辞書とグラフィックスステート

## 1. 概要

ページ辞書内の `/Resources` 辞書は、コンテンツストリーム内で使用される**ローカルな識別名**と、ファイル内の任意の場所に存在する**グローバルなインダイレクトオブジェクト**とを紐づける**ローカルなルックアップテーブル**として機能する。

## 2. リソース辞書の構造

```
<< /Resources
   << /Font << /F1 6 0 R /F2 7 0 R >>
      /XObject << /Im0 8 0 R /Fm1 9 0 R >>
      /ExtGState << /GS1 10 0 R >>
      /ColorSpace << /CS1 11 0 R >>
      /Pattern << /P1 12 0 R >>
      /Shading << /SH1 13 0 R >>
      /ProcSet [/PDF /Text /ImageB /ImageC /ImageI]
   >>
>>
```

## 3. リソースカテゴリ詳細

### 3.1 /Font（フォント辞書）

コンテンツストリーム内のフォント識別名を、実際のフォント辞書オブジェクトにマッピングする。

```
/Font << /F1 6 0 R    % Tf オペレータで "/F1" として参照
         /F2 7 0 R    % Tf オペレータで "/F2" として参照
         /TT0 15 0 R  % 命名規則は任意
      >>
```

#### コンテンツストリームでの使用例
```
BT
/F1 12 Tf        % /F1 フォントを12ポイントで使用
(Hello) Tj
ET
```

#### 実装仕様
```
名前解決フロー:
  1. ストリーム内で "/F1" を検出
  2. ページの /Resources → /Font 辞書を参照
  3. /F1 キーに対応するインダイレクト参照 (6 0 R) を取得
  4. オブジェクト6を解決してフォント辞書を取得
  5. フォント辞書からフォントプログラム、エンコーディング等を読み込み
```

### 3.2 /XObject（外部オブジェクト辞書）

再利用可能なグラフィカルコンテンツをマッピングする。

```
/XObject << /Im0 8 0 R     % ラスタ画像
            /Fm1 9 0 R     % フォームXObject
         >>
```

#### XObjectのサブタイプ

| サブタイプ | 説明 |
|:-----------|:-----|
| `/Image` | ラスタ画像（JPEG、PNG等の画像データ） |
| `/Form` | カプセル化された再利用可能なベクター図形ストリーム |
| `/PS` | PostScriptフラグメント（レガシー、非推奨） |

#### Image XObject の主要エントリ

| キー | 型 | 説明 |
|:-----|:---|:-----|
| `/Type` | 名前 | `/XObject` |
| `/Subtype` | 名前 | `/Image` |
| `/Width` | 整数 | 画像の幅（ピクセル） |
| `/Height` | 整数 | 画像の高さ（ピクセル） |
| `/ColorSpace` | 名前 or 配列 | 色空間（`/DeviceRGB`、`/DeviceGray` など） |
| `/BitsPerComponent` | 整数 | 1コンポーネントあたりのビット数（1, 2, 4, 8, 16） |
| `/Filter` | 名前 or 配列 | 圧縮フィルタ（`/DCTDecode` = JPEG など） |
| `/SMask` | ストリーム | ソフトマスク（アルファチャンネル） |
| `/Mask` | ストリーム or 配列 | ハードマスクまたはカラーキーマスク |

#### Form XObject の主要エントリ

| キー | 型 | 説明 |
|:-----|:---|:-----|
| `/Type` | 名前 | `/XObject` |
| `/Subtype` | 名前 | `/Form` |
| `/BBox` | 配列 | バウンディングボックス `[llx lly urx ury]` |
| `/Matrix` | 配列 | フォーム空間からユーザー空間への変換行列（デフォルト: 単位行列） |
| `/Resources` | 辞書 | フォーム内で使用されるリソース（独自のスコープ） |

#### コンテンツストリームでの使用例
```
q                     % グラフィックスステート保存
100 0 0 100 50 700 cm % 変換行列設定（位置・サイズ指定）
/Im0 Do               % XObject "Im0" を描画
Q                     % グラフィックスステート復元
```

### 3.3 /ExtGState（拡張グラフィックスステート辞書）

単一のオペレータでは表現できない複雑な描画状態を辞書として定義する。

```
/ExtGState << /GS1 10 0 R >>
```

#### 拡張グラフィックスステートの主要エントリ

| キー | 型 | 説明 | デフォルト |
|:-----|:---|:-----|:-----------|
| `/Type` | 名前 | `/ExtGState` | - |
| `/LW` | 実数 | 線の太さ | 1.0 |
| `/LC` | 整数 | 線端の形状（0=Butt, 1=Round, 2=Square） | 0 |
| `/LJ` | 整数 | 線接合の形状（0=Miter, 1=Round, 2=Bevel） | 0 |
| `/ML` | 実数 | マイター限界値 | 10.0 |
| `/D` | 配列 | 破線パターン `[[dash_array] phase]` | `[[] 0]` |
| `/RI` | 名前 | レンダリングインテント | `/RelativeColorimetric` |
| `/OP` | ブール | ストロークのオーバープリント | false |
| `/op` | ブール | フィルのオーバープリント | false |
| `/OPM` | 整数 | オーバープリントモード | 0 |
| `/Font` | 配列 | `[font_ref size]` フォントとサイズ | - |
| `/ca` | 実数 | フィルの不透明度（0.0〜1.0） | 1.0 |
| `/CA` | 実数 | ストロークの不透明度（0.0〜1.0） | 1.0 |
| `/BM` | 名前 or 配列 | ブレンドモード | `/Normal` |
| `/SMask` | 辞書 or 名前 | ソフトマスク辞書 or `/None` | `/None` |
| `/AIS` | ブール | アルファソースフラグ | false |
| `/TK` | ブール | テキストノックアウトフラグ | true |

#### ブレンドモード一覧

| 名前 | 説明 |
|:-----|:-----|
| `/Normal` | 通常（デフォルト） |
| `/Multiply` | 乗算 |
| `/Screen` | スクリーン |
| `/Overlay` | オーバーレイ |
| `/Darken` | 比較（暗） |
| `/Lighten` | 比較（明） |
| `/ColorDodge` | 覆い焼きカラー |
| `/ColorBurn` | 焼き込みカラー |
| `/HardLight` | ハードライト |
| `/SoftLight` | ソフトライト |
| `/Difference` | 差の絶対値 |
| `/Exclusion` | 除外 |

#### コンテンツストリームでの使用例
```
/GS1 gs               % 拡張グラフィックスステート "GS1" を適用
```

### 3.4 /ColorSpace（色空間辞書）

デバイス非依存の色空間定義をマッピングする。

```
/ColorSpace << /CS1 [/ICCBased 11 0 R]
               /CS2 [/Indexed /DeviceRGB 255 12 0 R]
            >>
```

#### 色空間の種類

| カテゴリ | 色空間名 | 説明 |
|:---------|:---------|:-----|
| デバイス依存 | `/DeviceGray` | グレースケール（1コンポーネント） |
| デバイス依存 | `/DeviceRGB` | RGB色空間（3コンポーネント） |
| デバイス依存 | `/DeviceCMYK` | CMYK色空間（4コンポーネント） |
| CIEベース | `/CalGray` | キャリブレーテッドグレースケール |
| CIEベース | `/CalRGB` | キャリブレーテッドRGB |
| CIEベース | `/Lab` | CIE L*a*b*色空間 |
| CIEベース | `/ICCBased` | ICCプロファイルに基づく色空間 |
| 特殊 | `/Indexed` | インデックスカラー（パレット）色空間 |
| 特殊 | `/Pattern` | パターン色空間 |
| 特殊 | `/Separation` | 特色（スポットカラー） |
| 特殊 | `/DeviceN` | 複数の特色を組み合わせた色空間 |

### 3.5 /Pattern（パターン辞書）

タイリングパターンやシェーディングパターンを定義する。

| パターンタイプ | 値 | 説明 |
|:---------------|:---|:-----|
| タイリングパターン | 1 | 繰り返し描画される小さな図形の定義 |
| シェーディングパターン | 2 | グラデーション等のスムーズな色変化 |

### 3.6 /Shading（シェーディング辞書）

グラデーション描画を定義する。

| シェーディングタイプ | 値 | 説明 |
|:---------------------|:---|:-----|
| 関数ベース | 1 | 2D関数による色定義 |
| 軸方向 | 2 | 2点間の線形グラデーション |
| 放射状 | 3 | 2つの円間の放射グラデーション |
| フリーフォームメッシュ | 4 | 自由形式のメッシュグラデーション |
| ラティスフォームメッシュ | 5 | 格子形式のメッシュグラデーション |
| クーンズパッチメッシュ | 6 | クーンズパッチによるメッシュ |
| テンソル積パッチメッシュ | 7 | テンソル積パッチによるメッシュ |

### 3.7 /ProcSet（手続きセット配列）

```
/ProcSet [/PDF /Text /ImageB /ImageC /ImageI]
```

| 名前 | 説明 |
|:-----|:-----|
| `/PDF` | PDF基本描画オペレータ |
| `/Text` | テキスト描画オペレータ |
| `/ImageB` | モノクロ画像オペレータ |
| `/ImageC` | カラー画像オペレータ |
| `/ImageI` | インデックスカラー画像オペレータ |

**注意**: モダンなPDFプロセッサでは実質的に無視されるが、仕様上の互換性のために生成時に含めることが推奨される。

## 4. グラフィックスステートマシン

### 4.1 ステートの構成要素

ライブラリが管理すべきグラフィックスステートの完全な一覧。

| パラメータ | デフォルト値 | 設定方法 |
|:-----------|:-------------|:---------|
| CTM（カレント変換行列） | 単位行列 | `cm` オペレータ |
| クリッピングパス | ページ全体 | `W` / `W*` オペレータ |
| 色空間（ストローク） | `/DeviceGray` | `CS` / `RG` / `K` |
| 色空間（フィル） | `/DeviceGray` | `cs` / `rg` / `k` |
| 色（ストローク） | 黒 | `SC` / `SCN` / `RG` / `K` / `G` |
| 色（フィル） | 黒 | `sc` / `scn` / `rg` / `k` / `g` |
| 線の太さ | 1.0 | `w` オペレータ |
| 線端の形状 | 0 (Butt) | `J` オペレータ |
| 線接合の形状 | 0 (Miter) | `j` オペレータ |
| マイター限界値 | 10.0 | `M` オペレータ |
| 破線パターン | 実線 | `d` オペレータ |
| レンダリングインテント | RelativeColorimetric | `ri` オペレータ |
| フラットネス許容値 | 1.0 | `i` オペレータ |
| 拡張ステート | - | `gs` オペレータ |
| テキストステート | 下記参照 | テキストオペレータ |

### 4.2 テキストステート

テキスト描画に関する追加のステートパラメータ。

| パラメータ | デフォルト値 | 設定オペレータ | 説明 |
|:-----------|:-------------|:---------------|:-----|
| 文字間隔 | 0 | `Tc` | 文字間の追加スペース |
| 単語間隔 | 0 | `Tw` | 単語間の追加スペース（スペース文字のみに適用） |
| 水平スケーリング | 100 | `Tz` | テキストの水平方向の拡大率（%） |
| テキストリーディング | 0 | `TL` | 行間隔 |
| フォント / サイズ | なし | `Tf` | 使用フォントとサイズ |
| テキストレンダリングモード | 0 (Fill) | `Tr` | テキストの描画方法 |
| テキストライズ | 0 | `Ts` | ベースラインからの垂直オフセット |

#### テキストレンダリングモード

| 値 | 説明 |
|:---|:-----|
| 0 | フィル（塗りつぶし） |
| 1 | ストローク（輪郭） |
| 2 | フィル＋ストローク |
| 3 | 不可視 |
| 4 | フィル＋クリッピングパスに追加 |
| 5 | ストローク＋クリッピングパスに追加 |
| 6 | フィル＋ストローク＋クリッピングパスに追加 |
| 7 | クリッピングパスに追加のみ |

### 4.3 ステートスタックの実装

```
struct GraphicsState {
    ctm: Matrix,
    clipping_path: Path,
    stroke_color_space: ColorSpace,
    fill_color_space: ColorSpace,
    stroke_color: Color,
    fill_color: Color,
    line_width: f64,
    line_cap: LineCap,
    line_join: LineJoin,
    miter_limit: f64,
    dash_pattern: DashPattern,
    rendering_intent: RenderingIntent,
    flatness: f64,
    // テキストステート
    char_spacing: f64,
    word_spacing: f64,
    horizontal_scaling: f64,
    text_leading: f64,
    text_font: Option<Font>,
    text_font_size: f64,
    text_rendering_mode: TextRenderingMode,
    text_rise: f64,
    // 透明度関連
    stroke_alpha: f64,
    fill_alpha: f64,
    blend_mode: BlendMode,
    soft_mask: Option<SoftMask>,
}

struct GraphicsStateStack {
    stack: Vec<GraphicsState>,
    current: GraphicsState,
}

impl GraphicsStateStack {
    fn save(&mut self) {         // q オペレータ
        self.stack.push(self.current.clone());
    }

    fn restore(&mut self) {      // Q オペレータ
        if let Some(state) = self.stack.pop() {
            self.current = state;
        }
    }
}
```
